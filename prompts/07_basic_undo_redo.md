# 07 — Basic Undo / Redo

## Goal
Add a practical first-pass undo/redo system for editing workflows.

## Requirements
- Support undo/redo for the main editing actions completed so far.
- Provide visible UI controls or keyboard shortcuts if practical.
- Keep the implementation understandable and not over-engineered.
- Document any limitations in STATUS.md.

## Deliverables
- Functional undo/redo support
- UI hooks for using it
- Clean enough architecture to extend later

## Notes
- Snapshot-based history is acceptable for MVP if performance remains reasonable.
